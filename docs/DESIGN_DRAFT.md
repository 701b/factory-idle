# Factory Idle - 게임 설계 문서

## 전체 구조 (큰 그림)

### 1단계: 행성 식민지화 (기반 구축) ← 현재 구체화
- 채굴 → 가공 → 조립으로 생산 체인
- 전력 등 인프라 구축
- 연구로 새 기계/레시피 해금 및 효율 업그레이드
- idle 핵심: 자동 생산, 오프라인 수익

### 2단계: 로켓 발사 & 상업 (나중에 구체화)
- 로켓 건조가 큰 마일스톤
- 우주정거장을 통한 제품 판매
- 행성별 고정 수요량, 초과분 안팔림
- 제품별 고정 가격 (가공 단계 깊을수록 비쌈)
- 돈으로 구매 전용 자원 구입 가능

### 3단계: 우주선 & 직접 거래 (나중에 구체화)
- 우주선 제작
- 다른 행성과 직접 거래
- 더 넓은 무역 네트워크

---

## 설계 원칙

- **단순함 우선**: 물류(컨베이어, 밸런서 등) 제거. 기계를 짓고 수량만 관리하면 자동 생산
- **명확한 진행감**: 해금 순서가 선형적, "다음에 뭘 해야 하는지" 항상 알 수 있음
- **idle 적합성**: 접속하지 않아도 자원이 쌓이고, 접속 시 의미 있는 선택만

### 1-2. 생산 체인

```
[채굴기] ─── 철 원석 ──→ [분쇄기] ──→ [제련기] ──→ 철판
[채굴기] ─── 구리 원석    ──→ [분쇄기] ──→ [제련기] ──→ 구리봉
[채굴기] ─── 석탄 원석     ──→ [분쇄기] ──→ [제련기] ──→ 코크스
[수집기] ─── 바이오매스       (분쇄/제련 불필요)

                철판 ──────────→ [조립기] ──→ 철 프레임
        철판 + 구리봉 ──────────→ [조립기] ──→ 회로기판
        프레임 + 구리봉 ─────────→ [조립기] ──→ 모터

        프레임 + 회로기판 ─────→ [조립기] ──→ 빨간 과학팩
  모터 + 회로기판 + 철판 ─────→ [조립기] ──→ 초록 과학팩
  과학팩(빨+초) + 모터 + 프레임 → [조립기] ──→ 파란 과학팩
```

### 1-3. 자원 배분 규칙

범용 기계(분쇄기, 제련기, 조립기)는 여러 레시피를 처리할 수 있다. 자원 배분 방식:

- **기계별 레시피 고정**: 각 기계 인스턴스는 하나의 레시피만 담당
  - 예: 분쇄기 3대 → 철 2대, 구리 1대로 배정
  - 플레이어가 기계 건설 시 레시피 선택 (이후 변경 가능)
- **조립기도 동일**: 조립기 건설 시 "철 프레임", "회로기판" 등 레시피 지정
- **의도**: "수량만 관리"하되 "어떤 레시피에 몇 대를 배정할지"가 핵심 선택

```
분쇄기 건설 UI:
┌─────────────────────────────┐
│ 분쇄기 건설 (비용: 철판x10) │
│                             │
│ 레시피 선택:                │
│ ○ 철 분쇄  (현재 2대) │
│ ○ 구리 분쇄     (현재 1대) │
│ ○ 석탄 분쇄      (현재 0대) │
│                             │
│ [건설] [x5] [x10] [Max]    │
└─────────────────────────────┘
```

### 2-2. 기계 레벨업

- 레벨당: 생산 속도 +10%, 전력 소비 +5%
- 최대 레벨: 20 (연구로 상한 해금)
- 비용: `baseCost * (level * 2)`

### 2-3. 비용 배율 공식

```
n번째 기계 비용 = baseCost * costMultiplier^(n-1)
```

### 2-4. 일괄 구매

- 기계 건설 시 수량 선택: **x1 / x5 / x10 / Max**
- Max = 현재 자원으로 구매 가능한 최대 수량
- 일괄 비용은 기하급수적 비용을 합산하여 표시
- 레벨업도 동일한 일괄 구매 UI 적용

### 2-5. 기계 해체 & 레시피 변경

- **레시피 변경**: 범용 기계(분쇄기/제련기/조립기)는 언제든 레시피 재배정 가능 (무료)
  - 기계 패널에서 기존 기계 클릭 → 레시피 드롭다운 변경
- **기계 해체**: 기계 1대 해체 시 건설 비용의 50% 자원 환불
  - 해체는 즉시 완료, 확인 다이얼로그 표시
  - 일괄 해체 미지원 (실수 방지)
- **의도**: 잘못된 배치를 부담 없이 수정하되, 무한 재활용은 방지

## 3. 전력 시스템

전력은 **저장 불가**. 생산 >= 소비여야 정상 가동.

### 3-1. 발전기

| 발전기 ID | 이름 | 연료 | 발전량 | 비용 | 해금 조건 |
|-----------|------|------|--------|------|----------|
| `biomass_burner` | 바이오매스 버너 | 바이오매스 1/s | 20W | 바이오매스x20 | 시작 |
| `coke_boiler` | 코크스 보일러 | 코크스 0.5/s | 60W | 철판x50 | 연구 |
| `turbine_generator` | 터빈 발전기 | 코크스 1/s | 200W | 프레임x20, 모터x10 | 연구 |

### 3-2. 전력 부족 시

```
전력비율 = min(1, totalProduction / totalConsumption)
실제생산속도 = 기본생산속도 * 전력비율
```

## 4. 연구 트리

### 4-1. 시스템

연구는 **2단계**로 나뉜다:

**원시 연구** (초반, 연구 서버 불필요)
- 자원을 직접 소비하여 즉시 완료 (클릭 한 번)
- 조립기 해금 전까지의 기초 기술을 다룸
- "수동 클릭은 초반 튜토리얼 느낌"과 일치

**과학 연구** (중반~, 연구 서버 필요)
- 연구 서버가 과학팩을 소비하여 자동 진행
- 한 번에 1개 연구만 진행 가능
- **연구 대기열**: 최대 3개까지 다음 연구를 예약 가능
  - 현재 연구 완료 시 대기열의 다음 연구 자동 시작
  - 오프라인 중에도 대기열 순서대로 진행
  - 대기열이 비어있으면 연구 서버 유휴 상태

## 5. Idle 메카닉

### 5-1. 게임 루프 (틱 간격: 100ms)

```
1. 전력 밸런스 계산
2. 전력비율 산출
3. 접속 부스트 배율 계산 (활성 시 2배 또는 3배)
4. 각 기계 progress += (delta / duration) * 전력비율 * 보너스배율 * 부스트배율
5. progress >= 1 이면 입력 확인 + 출력 저장 한도 확인 → 소비/생산 또는 대기
6. 연구 서버: 과학팩 소비 → 연구 진행
7. 업적 조건 체크 (이벤트 기반)
8. UI 갱신 (requestAnimationFrame throttle)
```

### 5-2. 오프라인 수익

- 최대 8시간
- 마지막 접속 시점의 "초당 순생산량" 스냅샷 기반
- 오프라인 효율 50% (연구로 상향 가능)

### 5-3. 게임 시작 초기 상태

게임 시작 시 다음 기계가 무료로 제공됨 (순환 의존 방지):

| 기계 | 수량 | 레시피 |
|------|------|--------|
| 바이오매스 수집기 | 1 | - |
| 바이오매스 버너 | 1 | - |
| 철 채굴기 | 1 | - |
| 분쇄기 | 1 | 철 분쇄 |
| 제련기 | 1 | 철 제련 |

- 이 상태에서 즉시 철판 생산 시작 → 추가 기계 건설 가능
- 초기 전력: 버너 20W 생산, 채굴기 5W + 분쇄기 10W + 제련기 15W = 30W 소비 → **전력 부족**
- 플레이어의 첫 행동: 바이오매스 수집기/버너를 추가 건설하여 전력 확보

### 5-4. 자동화 단계

- 게임 시작 5분 이내 완전 자동화 도달 목표
- 수동 클릭은 원시 연구(Phase A) + 기초 과학(B1) 해금 시에만

### 5-5. 접속 보너스

- 1시간 이상 미접속 후 복귀 시 "작업 부스트" 활성화
- 부스트 효과: 모든 생산 속도 2배, 지속시간 = min(미접속 시간 × 0.1, 30분)
- UI에 부스트 남은 시간 표시 (상단 바)
- 의도: 돌아올 동기 부여 + "접속 시 의미 있는 경험"

### 5-6. 프레스티지 (1단계 내)

- 로켓 과학 완료 시 "새 행성 식민지화" 가능
- 모든 자원/기계/연구 초기화
- **재개발 포인트(RP)** 획득: `floor(10 × (1 + 프레스티지 횟수 × 0.5))`
- 업적 보너스, 계약 달성 기록은 초기화되지 않음

### 5-6a. 행성 선택 (프레스티지 차별화)

프레스티지 시 다음 행성을 선택. 행성마다 환경이 달라 매 회차 다른 전략을 요구:

| 행성 | 특성 | 전략적 영향 |
|------|------|------------|
| 테라노바 (기본) | 균형잡힌 환경 | 표준 진행 |
| 페로시아 | 철 매장량 2배, 구리 절반 | 철판 과잉 → 회로 병목 |
| 이그니스 | 석탄 풍부, 바이오매스 희소 | 초반 전력난, 코크스 보일러 우선 전략 |
| 크리스탈리아 | 모든 원석 채굴 느림(-30%), 가공 속도 빠름(+50%) | 소수 정예 채굴기 + 대량 가공 |
| 오비탈 | 태양광 패널 사용 가능(무료 전력 50W), 저장 한도 절반 | 전력 여유, 창고 중요 |

- 첫 플레이는 항상 테라노바
- 2회차부터 행성 선택 가능 (해금된 행성 중)
- 행성 해금: 특정 프레스티지 횟수 도달 시
  - 2회차: 페로시아 or 이그니스 (택 1)
  - 3회차: 나머지 1개
  - 4회차: 크리스탈리아
  - 5회차: 오비탈

### 5-7. 재개발 포인트 상점

RP로 구매하는 영구 업그레이드 (프레스티지 후에도 유지):

| 업그레이드 | 비용 | 효과 | 최대 |
|-----------|------|------|------|
| 생산 가속 | 5 RP | 모든 생산 속도 +15% | 5회 |
| 시작 자원 | 3 RP | 게임 시작 시 철판 50, 구리봉 20 지급 | 3회 |
| 연구 건너뛰기 | 8 RP | 원시 연구(Phase A) 전체 자동 완료 | 1회 |
| 저장 확장 | 4 RP | 기본 저장 한도 +50% | 3회 |
| 전력 효율 | 6 RP | 모든 전력 소비 -10% | 3회 |
| 오프라인 강화 | 7 RP | 오프라인 효율 +15% | 3회 |
| 부스트 강화 | 10 RP | 접속 부스트 배율 +1x | 2회 |

**프레스티지 3회차 이후 해금되는 고급 업그레이드:**

| 업그레이드 | 비용 | 효과 | 최대 | 해금 조건 |
|-----------|------|------|------|----------|
| 자동 건설 | 15 RP | 계약 완료 시 가장 부족한 기계 1대 자동 건설 | 1회 | 프레스티지 3회 |
| 연구 가속 | 12 RP | 연구 서버 속도 +25% | 3회 | 프레스티지 3회 |
| 계약 보상 강화 | 8 RP | 계약 보상 +50% | 3회 | 프레스티지 4회 |
| 행성 마스터리 | 20 RP | 방문했던 행성의 특성 보너스 일부를 다른 행성에서도 유지 | 1회 | 프레스티지 5회 |

## 6. 계약 시스템

### 6-1. 개요

행성 본부에서 자원 납품 요청이 들어온다. 짧은 목표 사이클로 "한 번만 더" 심리를 유발.

- 항상 **3개의 활성 계약** 존재
- 계약 = "자원 X를 N개 납품" (재고에서 자동 차감)
- 완료 시 보상 지급 + 새 계약 자동 생성
- 오프라인 중에도 재고가 쌓이면 계약 자동 완료

### 6-2. 계약 등급

| 등급 | 난이도 | 보상 예시 | 해금 조건 |
|------|--------|----------|----------|
| 일반 | 현재 생산량 5~15분 분량 | 임시 부스트 (5분간 1.5배) | 시작 |
| 고급 | 현재 생산량 30분~1시간 분량 | RP 1~2점 | 과학 연구 시작 후 |
| 긴급 | 현재 생산량 5분 분량, 제한시간 10분 | RP 3점 + 저장 한도 임시 2배(30분) | 과학 연구 시작 후 |

### 6-3. 계약 생성 규칙

- 현재 생산 가능한 자원만 요구 (해금 안 된 자원 제외)
- 납품량 = 해당 자원의 초당 순생산량 × 목표 시간(초)
- 플레이어 진행도에 따라 자동 스케일링
- 같은 자원이 연속 출현하지 않도록 다양성 보장

### 6-4. 설계 의도

- **Short-term hook**: "이 계약만 끝내면..." → 접속 유지
- **접속 보상 느낌**: 돌아오면 계약 1~2개가 자동 완료되어 있음
- **RP 보조 수급**: 프레스티지 외에도 소량의 RP 획득 가능 → 장기 플레이 동기
- 연구 트리(해금 순서)와 겹치지 않음: 계약은 "생산량 목표", 연구는 "기술 해금"

## 10. 난이도 곡선 & 진행 목표

### 10-1. 예상 진행 속도

플레이어의 예상 진행 속도 (첫 플레이, 테라노바 기준):

| 시간대 | 마일스톤 | 플레이어 행동 |
|--------|---------|-------------|
| 0~5분 | 전력 안정화 + 추가 채굴기 | 바이오매스 축적 → 버너/수집기 추가, 분쇄기/제련기 증설 |
| 5~15분 | 원시 연구 A1~A3 완료 | 철판 생산 안정화, 구리/석탄 해금 |
| 15~30분 | 원시 연구 A4~A6 완료 | 코크스 보일러 가동, 조립기 해금 |
| 30분~1시간 | 빨간 과학팩 생산, B1~B3 | 연구 서버 가동, 초록 과학팩 진입 |
| 1~2시간 | 터빈 발전기, 고급 조립 | 전력 안정화, 대규모 확장 |
| 2~4시간 | 파란 과학팩, B7~B9 | 최종 연구 체인 진행 |
| 4~8시간 | 로켓 과학 완료 | 첫 프레스티지 (테라노바 → 새 행성) |
| 2회차 | 2~4시간 만에 완료 | RP 보너스 + 새 행성 전략 |

### 10-2. 초반 밸런스 시뮬레이션 (0~5분)

```
[초기 상태]
수집기 1대: 바이오매스 +2/s
버너 1대:   바이오매스 -1/s → 전력 20W
채굴기 1대: 원석 +1/s        → 전력 -5W
분쇄기 1대: 원석→분쇄물 1/3s  → 전력 -10W
제련기 1대: 분쇄물→철판 1/4s  → 전력 -15W

전력: 20W 생산 / 30W 소비 → 전력비율 0.67
바이오매스: +2 - 1 = +1/s (순생산)
철판: 0.25 × 0.67 = 약 0.17/s

[1분 후] 바이오매스 60개, 원석 40개, 철판 10개
→ 버너 추가 건설 가능 (바이오매스x20) → 전력 40W
→ 수집기 추가 건설 가능 (바이오매스x10)
→ 전력비율 1.0 도달, 철판 0.25/s로 회복

[3분 후] 철판 약 30개
→ 분쇄기 추가(철판x10), 제련기 추가(철판x15)

[5분 후] 철판 약 50개 이상
→ A1 연구 가능 (원석x50)
→ 계약 1개 완료 가능 (짧은 보상 사이클 시작)
```

### 10-3. 밸런스 원칙

- **초반 5분**: 절대 막히지 않아야 함. 바이오매스→전력→철판 흐름이 자연스럽게 연결
- **중반**: 병목은 "자원"이지 "시간"이 아니어야 함. 기계 배치 최적화가 핵심
- **후반**: 전력 관리 + 과학팩 생산 체인 효율화가 도전
- **프레스티지**: 2회차는 1회차의 50~60% 시간, RP + 행성 특성으로 전략 변화

## 11. 수익화 모델

### 11-1. 원칙

- **비침습적**: 게임 진행을 차단하지 않음. 무과금으로 100% 클리어 가능

| 계약 완료 시 | 보상 2배 | 계약당 1회 |
